INFRASTRUKTURA I SERWER VPS1\. Specyfikacja i LokalizacjaTyp serwera: VPS (Virtual Private Server).System operacyjny: Linux ubuntu (wersja terminalowa)Lokalizacja: Serwer jest dostępny pod Twoją domeną maxcore.dev. (hosting ovh) Dokładna fizyczna lokalizacja z punktu widzenia aplikacji kluczowa jest jej wysoka dostępność i stały adres IP powiązany z subdomena (a subdomeny to n8n.maxcore.dev, admin.maxcore.dev, w przyszłości więcej zależy od projektów)2\. Konfiguracja Usług (Backend Stack)Nginx (Reverse Proxy): Główny "bramkarz" serwera. Przekierowuje ruch z portu 443 (HTTPS) na port 5678 (n8n). (admin.maxcore.dev)n8n (Self-hosted): Działa jako silnik automatyzacji i backend API. Pozwala na nielimitowane przetwarzanie workflowów bez opłat za subskrypcję w chmurze n8n. (n8n.maxcore.dev)Let's Encrypt: Automatyczne wystawianie i odnawianie certyfikatów SSL, zapewniające zieloną kłódkę w przeglądarce i bezpieczne połączenie.3\. Zastosowane ZabezpieczeniaSzyfrowanie SSL/TLS (HTTPS): Cała komunikacja (zdjęcia produktów, dane o zamówieniach) jest szyfrowana. Nikt w Twojej sieci lokalnej ani u dostawcy Internetu nie może przechwycić przesyłanych informacji.Izolacja Portów: Porty usług wewnętrznych (np. 5678 dla n8n) są zablokowane przez firewall. Dostęp do nich jest możliwy tylko przez Nginx, co drastycznie zmniejsza ryzyko ataku.Ochrona przed atakami DoS/Payload: W konfiguracji Nginx ustawiliśmy client\_max\_body\_size 50M. Dzięki temu serwer akceptuje duże zdjęcia z iPhone'a, ale blokuje próby wysyłania gigantycznych plików, które mogłyby zawiesić serwer.Reverse Proxy: Ukrywa strukturę Twojej sieci wewnętrznej przed światem zewnętrznym. Atakujący widzi tylko Nginx, a nie Twoje bazy danych czy workflowy.4\. Proces Deploymentu (CI/CD)GitHub Integration: Kod źródłowy jest wypychany do repozytorium mmaaaxxx/asystent-magazyniera.Automatyzacja aktualizacji: Na serwerze znajduje się autorski skrypt ./aktualizuj.sh. Jego zadaniem jest:Pobranie najnowszego kodu (git pull).Zbudowanie wersji produkcyjnej (npm run build).Przeładowanie plików statycznych w Nginx.Restart procesów (jeśli wymagany).Na serverze jest postawiony VPN (WireQuard) z punktami dostepu na laptopie i iphon13pro.SSH działa na porcie 55000STRUKTURA PLIKÓW NA VPSStruktura została zaprojektowana tak, aby oddzielić kod źródłowy (repozytorium) od plików serwowanych użytkownikom oraz konfiguracji systemowych.1\. Katalog Projektu (Frontend \& Scripts)Lokalizacja: ~/asystent-magazyniera/ (katalog domowy użytkownika)Plaintextasystent-magazyniera/├── .git/                 # Dane repozytorium GitHub├── node\_modules/         # Biblioteki zainstalowane przez npm├── public/               # Zasoby statyczne (ikony, manifest PWA)│   ├── icon.png          # Ikona aplikacji widoczna na iPhone│   └── manifest.json     # Konfiguracja instalacji PWA├── src/                  # Kod źródłowy aplikacji│   ├── App.tsx           # GŁÓWNA LOGIKA (formularze, API, OCR)│   ├── types.ts          # Definicje typów danych (Order)│   ├── index.tsx         # Punkt wejścia React│   └── services/         # Opcjonalne serwisy (np. geminiService.ts)├── dist/                 # FOLDER PRODUKCYJNY (generowany przez npm run build)│                         # To te pliki faktycznie widzi użytkownik w przeglądarce.├── package.json          # Lista zależności i skryptów (Vite, React, Axios)├── vite.config.ts        # Konfiguracja narzędzia budującego├── aktualizuj.sh         # SKRYPT AUTOMATYZACJI (git pull \&\& npm run build)└── .gitignore            # Pliki ignorowane przez Git2\. Konfiguracja Serwera NginxLokalizacja: /etc/nginx/ (wymaga uprawnień root/sudo)Plaintext/etc/nginx/├── sites-available/│   └── n8n.maxcore.dev   # Plik konfiguracyjny (Proxy do n8n i ścieżka do dist/)├── sites-enabled/│   └── n8n.maxcore.dev   # Link symboliczny aktywujący powyższą konfigurację└── nginx.conf            # Główna konfiguracja serwera (tu jest client\_max\_body\_size)3\. Środowisko n8n (Backend)W zależności od metody instalacji, Twoje dane n8n znajdują się w:Docker: ~/n8n\_data/ (zawiera bazę danych SQLite oraz pliki konfiguracyjne workflowów).npm/pm2: ~/.n8n/ (domyślny ukryty katalog z konfiguracją i historią wykonań).4\. Skrypty PomocniczeLokalizacja: ~/infra/ (lub bezpośrednio w folderze projektu)aktualizuj.sh: Kluczowy skrypt, który wykonuje:git pull origin main – pobranie zmian z GitHub.npm install – aktualizacja bibliotek.npm run build – wygenerowanie nowych plików do folderu dist/.sudo systemctl reload nginx – (opcjonalnie) odświeżenie serwera.
